# Anti-Slop Rules

AI slop is the visual average of generic generated pages. It often looks polished at a glance and forgettable after five seconds.

This skill avoids it through hard rules.

## Forbidden by default

- Purple or purple-blue gradients.
- Decorative emoji used as structural icons.
- Neon cyan, neon green, or neon magenta on black.
- GitHub-dark `#0D1117` as the main background.
- Sticky sidebar navigation or default side table of contents.
- Main report content hidden behind tabs.
- Rounded card walls with no hierarchy.
- SVG illustrations of generic people, robots, or objects.
- Long raw tables visible in the main flow.
- Reader-facing notes about style compliance.

## Preferred defaults

- Warm ivory background.
- Low-saturation clay accent.
- Serif headings and system sans body text.
- One wide aligned main column.
- Evidence summarized first, raw details folded.
- One visual detail worth saving, not decoration everywhere.
- Plain text labels instead of emoji labels.

## Final check

Before delivery, ask:

- Can a tired reader understand the answer in the first screen?
- Can another agent audit the same report from JSON without scraping prose?
- Are long details folded?
- Are IDs aligned between HTML and JSON?
- Is any visual element generic decoration instead of communication?
